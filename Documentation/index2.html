Funcionalidad de la Api:
La API de pagos de colegio permite a los usuarios realizar operaciones CRUD (Crear, Leer, Actualizar, Eliminar) en una base de datos de pagos.
modelos de datos a ingresar
  • Recibo: Numero de recibo generado para el pago
  • Fecha: Fecha de realizacion del pago
  • Nombre: Nombre completo del estudiante
  • Concepto: Descripción del concepto por el cual se realizó el pago (por ejemplo, matrícula, mensualidad, libros, etc.).
  • Valor: Monto del pago que realizo
  • Fecha de ingreso: Fecha y hora en que se ingresó el pago en la base de datos. Automatica No se ingresa pero se puede obtener
  • Fecha de actualizacion: Fecha y hora en que se actulizo el pago en la base de datos. Automatica No se ingresa pero se puede obtener
    • Operaciones de la api:
  • Crear un nuevo registro de pagos
  • Leer todos los registro de pagos existentes
  • Leer un registro de pagos existente
  • Actualizar un registro de pagos existente
  • Eliminar un registro de pago existente

    • Cada operación de la API requiere que el usuario proporcione información específica de la solicitud, como el identificador del pago o los datos del pago para crear o actualizar un registro. La API también devuelve información de respuesta detallada, incluyendo códigos de estado HTTP, mensajes de error y datos del pago.

    Estructura de Carpetas:
      App:
    • Controllers: contiene las rutas de los Endopoint en un archivo llamado pagoscontrollers.js
    • Helpers: contiene una funcion para errores en un archivo llamado handleError.js
    • Models: contiene el modelo de la peticion a la base de datos en un archivo llamado pagos.js
    • Routes: contiene dos archivos index.js que permite reciber todos los archivos de rutas y pagosroute.js que son las rutas de los endpoint
      Config:
    • contiene la conexion a mongo con el archivo mongo.js
  • .env.example: contiene el ejemplo de las variables de entorno
  • app.js: contiene el archivo principal de la conexion
    • Como se podria implementar JWT
    JWT (JSON Web Tokens) es un método popular para la autenticación y la autorización en aplicaciones web y APIs. para implementarlo en esta api deberias crear un nuevo modelo donde permitas el ingreso de usuarios y que dividas los permisos de la siguiente manera

  • Administrador: Puede hacer todas las peticiones a la base de datos
  • Estudiante: solo puede consultar un dato especifica
